# V3.1a Timing Metadata

## Added

- Flutter Android node now emits PCM sample-index timing metadata with saved events.
- Backend `/events` accepts optional timing metadata.
- `events` and `event_group_observations` store nullable timing fields.
- Event Fusion observations copy timing metadata from the saved event.
- Dashboard event group details show a compact Timing section.
- Supabase safe migration: `migrations/v3_1a_timing_metadata.sql`.

## Preserved

- AudioRecord continuous monitoring
- Existing WAV upload flow
- AI detection and upload modes
- GPS updates
- WebSocket dashboard updates
- V3.0 Event Fusion grouping behavior

## Not Included

- MP3 conversion
- Time Sync API changes
- TDOA calculation
- GCC-PHAT or cross-correlation
- Tracking or target marker logic changes
