# V3.0 Event Fusion Layer Release Notes

## 新增內容

- 新增 Event Fusion Service。
- 新增 `event_groups` 與 `event_group_observations` 的融合欄位。
- 新增 `GET /event-groups`。
- 新增 `GET /event-groups/{group_id}`。
- Dashboard 右側新增「事件群組」面板。
- 新增 `tools/test_event_fusion.py`。

## 行為

- `/events` request body 不變。
- 原始事件仍寫入 `events`。
- 成功寫入原始事件後，後端會嘗試將事件加入 Event Group。
- 同 label 且時間相差 3 秒內的事件會被視為同一組。
- `node_count` 使用 distinct `device_id`。
- 重複 `event_id` 不會重複建立 observation。
- Event Fusion 失敗只記錄 log，不會讓 `/events` 失敗。

## 不包含

- 不修改 Flutter APP。
- 不修改 AI 推論流程。
- 不修改 WAV / GCS 上傳流程。
- 不新增 Time Sync。
- 不新增 TDOA。
- 不新增 Tracking。
- 不改既有遠端控制 API。

## 環境變數

可選：

- `EVENT_FUSION_WINDOW_SECONDS`，預設 `3`。

既有環境變數保持：

- `DATABASE_URL`
- `UPLOAD_TOKEN`
- `GCS_BUCKET_NAME`
- `GOOGLE_APPLICATION_CREDENTIALS_JSON`
- `GOOGLE_MAPS_API_KEY`

## 已知限制

- V3.0 只做事件群組，不做聲源定位。
- 群組時間依 `events.timestamp`，若手機時間不準，融合結果會受影響。
- 目前沒有背景排程器，舊 ACTIVE group 會在處理新事件時被關閉。
- 與 V3.1 聲源估測共用資料表，因此使用 `group_kind` / `observation_kind` 區分資料用途。
