# V3.1b Smart Audio Upload

## Summary

V3.1b adds smart audio packaging for accepted sound events:

- Full event audio is converted to MP3 for Dashboard playback and storage.
- A short PCM 16-bit WAV clip is extracted around the RMS peak for future
  GCC-PHAT / cross-correlation.
- Audio metadata is stored in `events` and copied into
  `event_group_observations`.

## Added

- `POST /upload-audio` now accepts primary MP3 or WAV audio.
- `POST /upload-tdoa-clip` uploads the short localization WAV clip.
- `GET /events/{event_id}/tdoa-clip-url` creates a signed playback URL for the
  short WAV clip.
- Dashboard Event Group observation detail shows:
  - primary audio format and size
  - source PCM size
  - compression saving percentage
  - encoding status
  - clip path availability and sample indexes
  - clip duration and source

## Compatibility

- Legacy WAV `audio_path` playback remains supported.
- Existing `/events` payloads remain valid.
- Existing `/upload-audio` WAV requests remain valid.
- New fields are nullable and safe for older data.

## Not Included

This release does not add Time Sync, TDOA computation, GCC-PHAT, tracking,
trajectory estimation, AI replacement, or Event Fusion grouping changes.
