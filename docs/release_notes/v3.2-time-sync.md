# V3.2 Time Sync

## Summary

V3.2 adds lightweight backend time synchronization for Flutter Android nodes.
Each node estimates clock offset and RTT using `/time-sync`, reports the latest
sync state through `/location-update`, and snapshots the current sync state into
each sound event.

## Added

- `GET /time-sync` returns server epoch milliseconds, ISO time, algorithm name,
  and RTT quality thresholds.
- `POST /location-update` accepts:
  - `time_sync_offset_ms`
  - `time_sync_rtt_ms`
  - `time_sync_quality`
  - `time_sync_at`
  - `last_time_sync_at`
- `device_status` stores latest per-device time sync state.
- `POST /events` accepts event-time sync snapshot fields:
  - `time_sync_version`
  - `time_sync_offset_ms`
  - `time_sync_rtt_ms`
  - `time_sync_quality`
  - `time_sync_synced_at_ms`
  - `time_sync_age_ms`
  - `corrected_arrival_time_ms`
- `event_group_observations` copies the same event-time sync snapshot so Event
  Fusion does not depend on later `device_status` updates.
- Dashboard node cards and map InfoWindow display sync quality, RTT, offset,
  and last sync time.
- Flutter app uses best-of-3 time-sync sampling and marks stale sync after the
  freshness window.
- Added backend and Flutter unit tests for sync metadata.

## Compatibility

- Existing APP versions can still call `/location-update` without sync fields.
- Existing `/events` payloads remain valid; all new event snapshot fields are
  nullable.
- Audio upload, TDOA clip upload, Dashboard playback, Event Fusion, and command
  polling remain unchanged.

## Not Included

This release does not add Time Sync calibration between devices beyond the
client-midpoint offset estimate. It does not implement GCC-PHAT, final TDOA
localization, or tracking.
